Aula SASS: @extend e @mixin

O SASS possui diversas funcionalidades que ajudam a organizar e reaproveitar o código. Vamos explorar as diretivas @extend e @mixin, que permitem reutilizar estilos de maneira eficiente e organizada.

1. @extend

A diretiva @extend é usada para compartilhar estilos entre seletores. Isso permite evitar duplicação de código, uma vez que diferentes elementos podem herdar os mesmos estilos base.

Exemplo


// SCSS
%botao-basico {
  padding: 10px 20px;
  border: none;
  border-radius: 5px;
  font-size: 16px;
}

.botao-primario {
  @extend %botao-basico;
  background-color: blue;
  color: white;
}

.botao-secundario {
  @extend %botao-basico;
  background-color: gray;
  color: black;
}

Resultado no CSS gerado:


.botao-primario, .botao-secundario {
  padding: 10px 20px;
  border: none;
  border-radius: 5px;
  font-size: 16px;
}

.botao-primario {
  background-color: blue;
  color: white;
}

.botao-secundario {
  background-color: gray;
  color: black;
}

Note que o uso de @extend reduz a repetição de código no CSS gerado.

2. @mixin

A diretiva @mixin permite criar blocos de código reutilizáveis que podem incluir parâmetros. Diferentemente do @extend, o @mixin copia o código para onde for chamado.

Exemplo


// SCSS
@mixin borda-arredondada($raio: 5px) {
  border: 1px solid #ccc;
  border-radius: $raio;
}

.card {
  @include borda-arredondada(10px);
  padding: 20px;
  background-color: #f9f9f9;
}

.alerta {
  @include borda-arredondada; // Usa o valor padrão (5px)
  padding: 15px;
  background-color: #ffcccc;
}

Resultado no CSS gerado:


.card {
  border: 1px solid #ccc;
  border-radius: 10px;
  padding: 20px;
  background-color: #f9f9f9;
}

.alerta {
  border: 1px solid #ccc;
  border-radius: 5px;
  padding: 15px;
  background-color: #ffcccc;
}

O @mixin é muito útil para estilos que precisam ser parametrizados, como tamanhos, cores ou espaçamentos.

3. Diferenças entre @extend e @mixin

Aspecto @extend @mixin
Reutilização Compartilha estilos entre seletores. Reutiliza blocos de código com ou sem parâmetros.
Herança Simula herança entre seletores. Não há herança; o código é copiado.
Flexibilidade Menos flexível, pois não aceita parâmetros. Mais flexível, com suporte a parâmetros.

Conclusão

Com o uso de @extend e @mixin, você pode organizar melhor o seu código SASS e evitar repetições desnecessárias. Escolha o mais adequado dependendo do contexto: use @extend para herança de estilos e @mixin para blocos reutilizáveis com maior flexibilidade.
